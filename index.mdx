## What is SlateDB?

SlateDB is an embedded storage engine built as a log-structured merge-tree (LSM-tree) that writes data directly to object storage like S3, GCS, Azure Blob Storage, and MinIO. Unlike traditional LSM-tree storage engines that use local disk, SlateDB leverages object storage to provide bottomless capacity, high durability, and easy replication.

## Key Features

<AccordionGroup>
  <Accordion title="Cloud Native Design">
    SlateDB is built from the ground up for object storage. It batches writes to reduce API costs, uses bloom filters and caching to optimize reads, and supports pluggable compaction that can run as a separate process.
  </Accordion>
  
  <Accordion title="Performance Optimized">
    Multi-tier caching (block cache, disk cache), compression (snappy, zlib, lz4, zstd), bloom filters, and configurable durability options to balance latency with persistence guarantees.
  </Accordion>
  
  <Accordion title="Advanced Features">
    Range queries, snapshots, checkpoints, merge operators, change data capture (CDC), and support for custom compaction filters and strategies.
  </Accordion>
  
  <Accordion title="Production Ready">
    Used by companies including Embucket, Gadget, Malstrom, Massive, Merklemap, Responsive, SQLync, and many more. Apache 2.0 licensed and part of the Commonhaus Foundation.
  </Accordion>
</AccordionGroup>

## Quick Example

Get started with SlateDB in minutes:

```rust
use slatedb::{Db, Error};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    // Open database with in-memory object store
    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());
    let db = Db::open("/tmp/my_db", object_store).await?;

    // Write data
    db.put(b"hello", b"world").await?;

    // Read data
    let value = db.get(b"hello").await?;
    assert_eq!(value, Some("world".into()));

    // Range queries
    let mut iter = db.scan(b"a"..=b"z").await?;
    while let Some(kv) = iter.next().await? {
        println!("{:?} = {:?}", kv.key, kv.value);
    }

    db.close().await?;
    Ok(())
}
```

## Why Object Storage?

Object storage provides several advantages for cloud-native applications:

- **Bottomless Capacity**: Scale storage independently without provisioning disks
- **High Durability**: Built-in replication across availability zones
- **Easy Replication**: Clone databases across regions with object storage replication
- **Cost Effective**: Pay only for what you store with automatic tiering

SlateDB mitigates object storage's higher latency and API costs through batching, caching, and compression.
