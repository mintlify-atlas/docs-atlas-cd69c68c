This guide will get you from zero to a working SlateDB application in under 5 minutes.

## Prerequisites

- Rust 1.70 or later
- Basic familiarity with async Rust and Tokio

## Step 1: Create a New Project

<Steps>
  <Step title="Create a new Rust project">
    ```bash
    cargo new slatedb-quickstart
    cd slatedb-quickstart
    ```
  </Step>

  <Step title="Add dependencies">
    Add SlateDB and Tokio to your `Cargo.toml`:
    
    ```toml Cargo.toml
    [dependencies]
    slatedb = "*"
    tokio = { version = "1", features = ["full"] }
    ```
  </Step>
</Steps>

## Step 2: Write Your First SlateDB Program

Replace the contents of `src/main.rs` with:

```rust src/main.rs lines
use slatedb::{Db, Error};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    // Create an in-memory object store
    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());
    
    // Open the database
    let db = Db::open("/tmp/quickstart", object_store).await?;
    
    // Write a key-value pair
    db.put(b"hello", b"world").await?;
    
    // Read it back
    let value = db.get(b"hello").await?;
    println!("Value: {:?}", value); // Some(b"world")
    
    // Close the database
    db.close().await?;
    
    Ok(())
}
```

## Step 3: Run Your Program

```bash
cargo run
```

You should see:

```
Value: Some(b"world")
```

Congratulations! You've just created your first SlateDB application.

## Next Steps: Core Operations

Now let's explore the core operations you'll use in SlateDB.

### Writing Data

```rust
// Put a single key-value pair
db.put(b"user:1", b"Alice").await?;

// Put with options (customize durability)
use slatedb::config::PutOptions;
db.put_with_options(
    b"user:2", 
    b"Bob",
    &PutOptions {
        await_durable: false, // Don't wait for flush
    }
).await?;

// Delete a key
db.delete(b"user:1").await?;
```

### Reading Data

```rust
// Get a single value
if let Some(value) = db.get(b"user:2").await? {
    println!("User: {:?}", value);
}

// Get with options
use slatedb::config::ReadOptions;
let value = db.get_with_options(
    b"user:2",
    &ReadOptions::default()
).await?;
```

### Batch Operations

```rust
use slatedb::WriteBatch;

// Create a batch
let mut batch = WriteBatch::new();
batch.put(b"user:1", b"Alice");
batch.put(b"user:2", b"Bob");
batch.put(b"user:3", b"Charlie");

// Write atomically
let handle = db.write_batch(batch).await?;
println!("Batch committed at seq: {}", handle.seqnum());
```

### Range Queries

```rust
// Scan all keys with prefix "user:"
let mut iter = db.scan(b"user:"..b"user:~").await?;
while let Some(kv) = iter.next().await? {
    println!("{:?} = {:?}", kv.key, kv.value);
}

// Scan with specific bounds
let mut iter = db.scan(b"user:1"..=b"user:2").await?;
while let Some(kv) = iter.next().await? {
    println!("{:?} = {:?}", kv.key, kv.value);
}

// Seek to a specific key
let mut iter = db.scan(..).await?;
iter.seek(b"user:2").await?;
if let Some(kv) = iter.next().await? {
    println!("Found: {:?}", kv.key);
}
```

## Using Real Object Storage

The in-memory store is great for testing, but let's use a real object store:

<Tabs>
  <Tab title="AWS S3">
    ```rust

    use std::sync::Arc;

    let object_store = Arc::new(
        AmazonS3Builder::new()

            .with_region("us-east-1")
            .build()?

    let db = Db::open("/mydb", object_store).await?;
    ```

## Complete Example

Here's a complete example showing common operations:

```rust
use slatedb::{Db, WriteBatch, Error};
use slatedb::object_store::memory::InMemory;
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    // Setup
    let object_store = Arc::new(InMemory::new());
    let db = Db::open("/tmp/quickstart", object_store).await?;

    // Write some data
    db.put(b"user:1:name", b"Alice").await?;
    db.put(b"user:1:email", b"alice@example.com").await?;
    db.put(b"user:2:name", b"Bob").await?;
    db.put(b"user:2:email", b"bob@example.com").await?;

    // Read a specific key
    let name = db.get(b"user:1:name").await?;
    println!("Name: {:?}", name);

    // Scan a range
    println!("\\nAll user:1 data:");
    let mut iter = db.scan(b"user:1:"..b"user:1:~").await?;
    while let Some(kv) = iter.next().await? {
        println!("  {:?} = {:?}", kv.key, kv.value);
    }

    // Batch update
    let mut batch = WriteBatch::new();
    batch.put(b"user:1:name", b"Alice Smith");
    batch.put(b"user:1:phone", b"555-1234");
    db.write_batch(batch).await?;

    // Delete
    db.delete(b"user:2:email").await?;

    // Flush to object storage
    db.flush().await?;

    // Close
    db.close().await?;

    println!("\\nDone!");
    Ok(())
}
```

## Troubleshooting

<Accordion title="Error: 'object_store' not found">
  SlateDB re-exports the `object_store` crate. Use `slatedb::object_store` instead of adding it as a separate dependency.
</Accordion>

<Accordion title="Async runtime errors">
  Make sure you're running your code within a Tokio runtime using `#[tokio::main]` or by manually creating a runtime.
</Accordion>

<Accordion title="Connection to object storage fails">
  - Verify your credentials are correct
  - Check that your bucket/container exists
  - Ensure network connectivity to the object storage service
</Accordion>
